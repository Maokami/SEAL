\doxysubsection{Detailed Description}
Class to store keyswitching keys. It should never be necessary for normal users to create an instance of \doxylink{classseal_1_1_k_switch_keys}{KSwitch\+Keys}. This class is used strictly as a base class for \doxylink{classseal_1_1_relin_keys}{Relin\+Keys} and \doxylink{classseal_1_1_galois_keys}{Galois\+Keys} classes.

\begin{DoxyParagraph}{Keyswitching}
Concretely, keyswitching is used to change a ciphertext encrypted with one key to be encrypted with another key. It is a general technique and is used in relinearization and Galois rotations. A keyswitching key contains a sequence (vector) of keys. In \doxylink{classseal_1_1_relin_keys}{Relin\+Keys}, each key is an encryption of a power of the secret key. In \doxylink{classseal_1_1_galois_keys}{Galois\+Keys}, each key corresponds to a type of rotation.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Thread Safety}
In general, reading from \doxylink{classseal_1_1_k_switch_keys}{KSwitch\+Keys} is thread-\/safe as long as no other thread is concurrently mutating it. This is due to the underlying data structure storing the keyswitching keys not being thread-\/safe.
\end{DoxyParagraph}
